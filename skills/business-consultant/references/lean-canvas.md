# Business Consultant — Lean Canvas Reference

## Lean Canvas Template
Use this structure when analyzing a business idea or validating product direction.

```
┌─────────────────┬──────────────────┬─────────────────┐
│   PROBLEM       │    SOLUTION      │ UNIQUE VALUE    │
│                 │                  │ PROPOSITION     │
│ Top 3 problems  │ Top 3 features   │                 │
│ users face      │ that solve them  │ Single clear    │
│                 │                  │ message         │
├─────────────────┤                  ├─────────────────┤
│ EXISTING        │                  │ UNFAIR          │
│ ALTERNATIVES    │                  │ ADVANTAGE       │
│                 │                  │                 │
│ What users do   │                  │ Can't easily    │
│ today           │                  │ be copied       │
├─────────────────┴──────────────────┴─────────────────┤
│   CUSTOMER SEGMENTS                                   │
│                                                       │
│   Early adopters → [specific person with the problem] │
│   Mainstream → [broader segment]                      │
├───────────────────────┬───────────────────────────────┤
│   CHANNELS            │   COST STRUCTURE              │
│                       │                               │
│   How we reach        │   Fixed costs: hosting, team  │
│   customers           │   Variable: per-user costs    │
├───────────────────────┼───────────────────────────────┤
│   KEY METRICS         │   REVENUE STREAMS             │
│                       │                               │
│   What we measure     │   How we make money           │
│   (activation,        │   (pricing model, tiers)      │
│   retention, revenue) │                               │
└───────────────────────┴───────────────────────────────┘
```

## Research-Backed Canvas Analysis
When filling out a Lean Canvas, the Business Consultant must:

1. **Problem**: Verify with real user pain evidence (forum posts, reviews, Reddit threads)
2. **Solution**: Does a simpler alternative already exist? Why wouldn't users use it?
3. **Unique Value Proposition**: Is this differentiation real and verifiable?
4. **Unfair Advantage**: Is this actually defensible, or will competitors copy it in 6 months?
5. **Customer Segments**: Who specifically is the early adopter? Name a real person type.
6. **Channels**: What's the CAC (customer acquisition cost) for each channel?
7. **Revenue**: What is the median willingness-to-pay for this segment? (Search for benchmarks)
8. **Cost Structure**: What is the unit economics at 100 customers? 1,000? 10,000?

## Red Flags (escalate to Claude)
- The "unique value proposition" is the same as 3+ existing competitors
- The "unfair advantage" is just "we'll build it better"
- The "customer segment" is "everyone" or "all small businesses"
- Revenue projections assume >5% market capture in year 1 (unrealistic)
