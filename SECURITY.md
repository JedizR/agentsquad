# Security

## API key handling

Your Gemini API key is stored only in `~/.agent-team/.env.team` on your local machine.

**How it's protected:**
- `chmod 600` is set by `setup.sh` — only your user account can read the file
- `.env.team` is listed in `~/.agent-team/.gitignore` — it won't be committed if you git-track that directory
- All project `.gitignore` files generated by `/init-agent-team` include `.env*` patterns

**What AgentSquad never does:**
- Sends your key to any server other than `generativelanguage.googleapis.com` (Google's API endpoint)
- Logs your key to files or stdout
- Stores your key anywhere outside `~/.agent-team/.env.team`

## If your API key is compromised

1. Revoke it immediately: [aistudio.google.com](https://aistudio.google.com) → **Manage API keys** → **Revoke**
2. Generate a new key in AI Studio
3. Update `~/.agent-team/.env.team`
4. Re-source: `source ~/.agent-team/.env.team`

If you're using a reserve key (`GEMINI_KEY_RESERVE_1`), revoke that too.

## Checking for accidental key exposure

If you think a key appeared in a commit:

```bash
# Search git history for your key (replace with actual key prefix)
git log -p | grep "AIzaSy"

# If found, rotate immediately and consider the key compromised
```

GitHub's secret scanning will flag Gemini keys in public repositories. If GitHub notifies you, rotate before doing anything else.

## Reporting vulnerabilities

If you find a security issue in AgentSquad (script injection, credential exposure risk, unsafe defaults), do not open a public issue.

Report privately via GitHub's [Security Advisories](https://github.com/JedizR/agentsquad/security/advisories/new) feature.

Include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

Expect a response within 48 hours. We'll coordinate a fix and credit you in the CHANGELOG when the patch is released.

## Scope

The following are **in scope** for vulnerability reports:
- Scripts that could expose API keys
- SKILL.md patterns that could cause agents to exfiltrate data
- `setup.sh` file permissions issues
- Any default behavior that weakens credential security

The following are **out of scope:**
- The security of Google's AI Studio API itself
- The behavior of third-party Gemini CLI versions
- Issues requiring physical access to your machine
